# GitHub Copilot向けインストラクション

## 基本原則

- **SPEC.mdを常に確認し、矛盾があれば問い合わせる。自己判断しない。**
- SPEC.mdに記載されていない仕様を勝手に追加しない。
- 実装前にSPEC.mdの該当セクションを読み直し、仕様に沿っているか確認する。
- 仕様に曖昧さがある場合は、実装を進める前にユーザーに確認する。

## 技術的制約

- KIFパーサーは `tsshogi` を使用する。独自パーサーを書かない。
- デスクトップフレームワークは Tauri v2 を使用する。Electronは使わない。
- フロントエンドは React + TypeScript + Vite。
- 音声再生は Web Audio API を使用する。

## コーディング規約

- TypeScript の strict モードを有効にする。
- テストは Vitest を使用する。
- 変数名・関数名は英語、コメントは日本語で書く。
- 将棋用語は SPEC.md の表記に従う（例: "攻め方" "玉方"）。

## ファイル構成

- SPEC.md の「ディレクトリ構成」セクションに従う。
- 新しいディレクトリやファイルを追加する場合は、SPEC.md を先に更新する。
